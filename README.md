# SEng-Coursework
The system maintains a database of doctors, patients and appointments

В тази курсова работа се проектира, създава, тества, документира и защитава система за записване на посещения при лични лекари. Със системата се взаимодейства чрез приложно-програмен интерфейс (API) по HTTP(S) протокола. 

## Функционалности
- Регистрация на лекар
- Регистрация на пациент
- Вход в системата
- Задаване на работно време на лекар
- Добавяне на временна промяна в работното време
- Добавяне на постоянна промяна в работното време
- Създаване на посещение
- Отмяна на посещение
- Преглед на посещенията на текущия потребител
- JWT оторизация
- Автоматизирани тестове с pytest

## Използвани технологии
- Python
- FastAPI
- SQLAlchemy
- PostgreSQL
- SQLite (за тестове)
- JWT
- Pytest

## Стартиране
1. Инсталиране на зависимостите:
   pip install -r requirements.txt

2. Създаване на PostgreSQL база данни:
   medical_booking

3. Конфигуриране на DATABASE_URL в app/database.py

4. Стартиране:
   uvicorn app.main:app --reload


## Тестове
pytest -v

### Правила
- Пациентът може да записва посещение само при своя личен лекар.
- Посещението трябва да е изцяло в работното време на лекаря.
- Посещението не трябва да попада в почивка.
- Посещението трябва да бъде създадено поне 24 часа предварително.
- Посещението не трябва да се застъпва с друго активно посещение при същия лекар.
- Посещението може да бъде отменено от пациента или лекаря.
- Отмяната е позволена само ако са останали поне 12 часа до началото.
- Лекарят може да има най-много една временна промяна на работното време.
- Постоянна промяна в работното време влиза в сила поне 7 дни след създаването.
